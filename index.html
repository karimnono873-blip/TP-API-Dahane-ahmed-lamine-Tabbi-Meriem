1. Objectif du TP et Rappels Théoriques

🎯 Objectif Principal : L'objectif de ce TP est de permettre aux étudiants de concevoir, programmer et valider un système automatisé piloté par un automate programmable industriel (API S7-300), en utilisant le logiciel TIA Portal. Cela inclut le passage du cahier des charges au GRAFCET, son implémentation en langages LADDER et IL (AWL), intégrant la gestion des séquences, des temporisations (TON) et la validation par simulation (S7-PLCSIM).

Rappel de cours : API S7-300 et Programmation

Le SIMATIC S7-300 est un automate modulaire. Son cycle de traitement (Scan Cycle) s'effectue en 3 étapes :
1. Lecture des entrées (Copie des états physiques %I dans la mémoire image PII).
2. Exécution du programme utilisateur (OB1, traitement des réseaux LADDER/IL).
3. Écriture des sorties (Transfert de la mémoire image PIQ vers les modules physiques %Q).

Langages normalisés (CEI 61131-3) utilisés :

  • LADDER (LAD) : Représentation graphique à contacts (schéma électrique).
  • Instruction List (IL / AWL) : Langage textuel de bas niveau, similaire à l'assembleur, optimisé pour les temps de cycle très courts. Opérandes : A (AND), O (OR), S (Set), R (Reset).

2.2 Synthèse du GRAFCET (Partie 1)

( 0 ) <-- État initial (Cuve au repos) | +-- [ CPT_LSL ] (Condition : Niveau bas atteint) | ( 1 ) ---> ACTION : Activer Pompe 1 (PP_1) | +-- [ CPT_LSH ] (Condition : Niveau haut atteint) | ( 2 ) ---> ACTION : Activer Pompe 1 (PP_1) + Lancer Temporisation T1 (10s) | +-- [ t / 2 / 10s ] (Condition : 10 secondes écoulées) | +-----> (Retour à l'étape 0, Arrêt de la pompe)

2.4 Programme IL (AWL) - Partie 1

// Traitement de la temporisation de stabilité
      CALL  T_STABILITE (TON)
       IN := %I0.1      // Capteur LSH
       PT := T#10s      // Temps de consigne
       Q  := %M0.1      // M_STABLE

// Logique d'activation de la pompe
      A     %I0.0       // LSL (Niveau bas)
      O     %Q4.0       // OU Maintien de la pompe
      AN    %M0.1       // ET NON (Stabilité de 10s atteinte)
      =     %Q4.0       // Sortie Pompe

3.2 GRAFCET Maître (Divergence en ET)

( X0 ) <-- État Initial (a0.f0.e0.s0.p) | +-- [ dcy ] | ( X1 ) ---> S_PLUS (Serrage) | +-- [ s1 ] | +=========+=================+ (Divergence en ET : Tâches Simultanées) | | ( X11 ) -> F_PLUS (Fraisage) ( X21 ) -> A_PLUS (Lamage) | | +-- [ f1 ] +-- [ a1 ] | | ( X12 ) -> F_MOINS ( X22 ) -> Temporisation t/X22/1s | | +-- [ f0 ] +-- [ Fin Tempo 1s ] | | ( X13 ) -> Attente ( X23 ) -> A_MOINS | | | +-- [ a0 ] | | | ( X24 ) -> Attente +=========+=================+ (Convergence en ET) | ( X3 ) ---> S_MOINS (Desserrage) | +-- [ s0 ] | ( X4 ) ---> E_PLUS (Éjection - Vérin Monostable)

3.3 Programme LADDER - Partie 2 (Extrait Synchronisation)

// Étape X1 : Serrage (S_PLUS)
      %M1.1 (X1)
-------| |---------------------------------------------------( S )-- %Q5.4 (S_PLUS)
                                                             ( R )-- %Q5.5 (S_MOINS)

// Divergence en ET (Activation simultanée X11 et X21)
      %M1.1 (X1)    %I2.1 (s1)
-------| |-----------| |-------------------------------------( S )-- %M1.2 (X2)
                                                             ( R )-- %M1.1 (X1)
                                                             ( S )-- %M2.1 (X11) // Lancement TF
                                                             ( S )-- %M3.1 (X21) // Lancement TL

// Étape X4 : Ejection (Monostable nécessite une bobine standard)
      %M1.4 (X4)
-------| |---------------------------------------------------( )---- %Q5.6 (E_PLUS)

3.4 Programme IL (AWL) - Partie 2 (Séquence Principale)

// Gestion des étapes du Grafcet Maître en List d'Instructions
// Transition X0 -> X1
      A     %M1.0       // Si étape X0 active
      A     %I1.0       // Et départ cycle (dcy)
      S     %M1.1       // Activer X1
      R     %M1.0       // Désactiver X0

// Action X1 (Serrage Bistable)
      A     %M1.1
      S     %Q5.4       // Set S_PLUS
      R     %Q5.5       // Reset S_MOINS

// Transition X1 -> Tâches parallèles
      A     %M1.1       // Si étape X1 active
      A     %I2.1       // Et capteur s1 actionné
      S     %M2.1       // Activer X11 (Fraisage)
      S     %M3.1       // Activer X21 (Lamage)
      R     %M1.1       // Désactiver X1

// Action X4 (Ejection Monostable)
      A     %M1.4       // Si étape X4 active
      =     %Q5.6       // Action maintenue E_PLUS (= Bobine standard)
